## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (Tailwind `gap-1`, `px-1`) |
| sm | 8px | Compact element spacing (Tailwind `gap-2`, `px-2`) |
| md | 16px | Default element spacing (Tailwind `gap-4`, `px-4`) |
| lg | 24px | Section padding (Tailwind `gap-6`, `px-6`) |
| xl | 32px | Layout gaps (Tailwind `gap-8`, `px-8`) |
| 2xl | 48px | Major section breaks (Tailwind `py-12`) |
| 3xl | 64px | Page-level spacing (Tailwind `py-16`) |

Exceptions:
- Score bar rows: `gap-2` (8px) between label, bar, and percentage — matches the tight inline layout confirmed in wireframe
- Score bar section container: `gap-2` (8px) between each of the 3 rows — keeps the trio visually grouped
- Bar track height: `h-2` (8px) — minimum grid value; fine, data-viz bar with sufficient tap affordance

_Source: 08-CONTEXT.md wireframe + DailyGem.tsx existing gap-6 section cadence_

## Component Inventory

### New: ScoreBreakdown (extracted sub-component)

**Decision basis:** CONTEXT.md Claude's Discretion — extract if bars are >20 lines; confirmed pattern is `FeedbackButtonGroup` as a separate file.

**File:** `frontend/app/profile/components/DailyGem/ScoreBreakdown.tsx`

**Props interface:**
```ts
interface ScoreBreakdownProps {
  breakdown: Record<string, number>; // e.g. { genre_sim: 0.72, novelty: 0.61, feedback_multiplier: 0.19 }
}
```

**Render contract:**
- If `breakdown` is empty (`Object.keys(breakdown).length === 0`), render `null` — no placeholder, no greyed bars, no message
- Render exactly 3 rows in this fixed order: Genre Match (`genre_sim`), Novelty (`novelty`), Feedback (`feedback_multiplier`)
- Each row layout: `flex items-center gap-2` containing:
  - Label: `text-sm text-gray-300 w-28 flex-shrink-0` — left-anchored, fixed width to align bars
  - Bar track: `flex-1 bg-gray-800 rounded-full h-2 overflow-hidden`
  - Bar fill: `bg-green rounded-full h-full` with `style={{ width: \`${roundedPct}%\` }}`
  - Percentage: `text-sm font-bold text-gray-300 w-9 text-right flex-shrink-0`
- Rounding formula: `Math.round(rawValue * 100 / 5) * 5` — rounds to nearest 5% per ROADMAP success criterion
- Container: `flex flex-col gap-2` — wraps all 3 rows

**Placement in DailyGem.tsx:** Directly after the `{explanation && <blockquote>}` block, before the `{track.preview_url && ...}` audio section.

**Interface update required in DailyGem.tsx:**
```ts
interface DailyGemResponse {
  track: GemTrack;
  explanation: string;
  score_breakdown: Record<string, number>; // add — defaults to {} from API
  date: string;
  cached: boolean;
}
```

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Score bar render on load | Bars appear immediately when `gem.score_breakdown` is non-empty — no animation; static width from data |
| Score bar empty state | `score_breakdown` is `{}` or missing → `ScoreBreakdown` returns `null` — no transition, no flash |
| MetricsStrip hit rate | Renders on `fetchMetrics()` completion — same loading/error path as existing stats (no new fetch) |
| MetricsStrip refresh | "Refresh stats" button re-fetches and re-renders `compound_hit_rate` with the same `refreshing` state guard |
| No hover / tooltip on bars | Bars are display-only — no click, no tooltip, no hover state. Read-only data visualization. |
