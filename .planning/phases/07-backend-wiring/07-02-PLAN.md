<objective>
Wire `DailyGem.was_saved` from the Spotify-library-save endpoint and expose `compound_hit_rate`
(`(was_liked OR was_saved) / total_gems`) in the metrics endpoint.

Purpose: Closes the feedback loop with a measurable compound success metric. A "hit" =
the user either liked the gem OR saved it to their Spotify library. Per D-12, denominator
matches `gem_acceptance_rate` formula (all gems, nulls count as misses) so the two metrics
stay comparable in the dashboard.

Output: Non-fatal `was_saved` update in `add_track_to_liked` (mirrors the `was_liked`
non-fatal pattern at views.py line 631-637); `compound_hit_rate` key added to
`get_recommendation_metrics` JSON response alongside the existing `gem_acceptance_rate`.
</objective>
