<objective>
Wire score-breakdown, taste-vector-snapshot, and deterministic explanation text into the
`get_daily_gem` view. Persist the four score fields at gem creation time via the existing
`DailyGem.objects.get_or_create` call, fix the three cached-return sites that currently
hardcode `score_breakdown: {}`, and add a module-level `_build_gem_explanation` pure helper.

Purpose: Closes the explainability loop for v1.1 â€” every fresh gem persists what it knew
when it picked, the cached branch surfaces that same data on repeat requests, and the
frontend (Phase 8) can render a real breakdown without re-running the engine.

Output: `_build_gem_explanation` helper, expanded `get_or_create` defaults dict, two
`gem.score_breakdown` reads at cached-return sites, one `gem.explanation` read at the
fresh-branch JsonResponse, and test classes covering each behavior.
</objective>
