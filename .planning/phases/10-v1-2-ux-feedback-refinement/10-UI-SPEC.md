### Phase 10 Color Changes (UI-02 and UI-03)

#### UI-02: Popularity Label Color Semantics

The existing `getPopularityColor` is semantically inverted for a hidden-gem app (green=high popularity is wrong). Phase 10 replaces it with:

| Popularity Score | Label | Tailwind Color Class | Semantic Rationale |
|-----------------|-------|---------------------|-------------------|
| < 40 | Hidden Gem | `text-green` (#1DB954) | Low-popularity tracks are desirable — green = good find |
| 40–69 | Rising | `text-yellow-400` | Mid-range — neither hidden gem nor mainstream |
| ≥ 70 | Mainstream | `text-gray-400` | High-popularity tracks are common — muted = less distinctive |

`text-green` is the project's custom Tailwind alias for #1DB954 (confirmed in tailwind.config.ts line 23). `text-yellow-400` and `text-gray-400` are standard Tailwind v3 classes — no config change required.

The `color` value from `getPopularityLabel` applies to BOTH the `TrendingUp` icon className AND the text label div. One call per artist card render, destructured as `const pop = getPopularityLabel(artist.popularity)`.

#### UI-03: Expanded Artist Panel Background

Replace `bg-gray-850` (non-existent class → renders transparent) with `bg-gray-800` (#1F2937) at `TopArtists.tsx` line 190. This restores the visible background for the expanded detail panel, matching the existing card background.

## Interaction Contracts

### SYNC-02: FeedbackButtonGroup syncedFeedback Prop

- Accept `syncedFeedback?: 'LIKE' | 'DISLIKE' | null` as optional prop
- `useEffect` watching `syncedFeedback`: call `setSelectedFeedback(syncedFeedback)` only when `syncedFeedback != null`
- Guard rule: if prop resets to `null`, do NOT clear user's manual selection — the null guard (`!= null`) is mandatory
- No API call fires on prop change — visual-only mirror
- Existing `handleSubmit` path fires only on explicit user click — unchanged

### SYNC-03: DailyGem aiSyncedFeedback State

- Add `const [aiSyncedFeedback, setAiSyncedFeedback] = useState<'LIKE' | 'DISLIKE' | null>(null)`
- `onFeedbackSubmitted` callback changes signature from `() => void` to `(interpretation: any) => void`
- Mapping: `interpretation?.overall_sentiment === 'positive'` → `setAiSyncedFeedback('LIKE')`, `=== 'negative'` → `setAiSyncedFeedback('DISLIKE')`, else no state change
- `<FeedbackButtonGroup>` receives `syncedFeedback={aiSyncedFeedback}` prop

### EVOLVE-01/02: CustomEvent Live Refresh

- `DailyGem.tsx` dispatches `new CustomEvent('songscope:new-gem')` after `setGem(data)` — fires on both initial load and force_new
- `ImprovementStory.tsx` listens via `window.addEventListener('songscope:new-gem', handler)` in a `useEffect` with cleanup via `removeEventListener`
- `fetchMetrics` must be a named function at component scope (not inside a useEffect) to avoid stale closure
- Both components are `"use client"` — no SSR hazard

### UI-01: Remove Refresh Stats Button

- Remove `<button>` element (MetricsStrip.tsx lines 89–95) entirely
- Remove `const [refreshing, setRefreshing] = useState(false)` (line 32)
- Remove `if (showRefreshing) setRefreshing(true)` (line 36) and `setRefreshing(false)` (line 45)
- Remove `showRefreshing = false` parameter from `fetchMetrics` signature
- Auto-load on mount (existing `useEffect` calling `fetchMetrics()`) unchanged
