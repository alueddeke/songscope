<objective>
Add the `overall_sentiment` field to the AI feedback interpretation contract (SYNC-01). This is the backend half of the AI-feedback-to-thumbs sync: the OpenAI prompt schema gains an `overall_sentiment` field, a rule instructs the model how to set it, and the fallback path always emits the key so the frontend (Plan 03) can rely on it being present.

Purpose: SYNC-03 in Plan 03 reads `interpretation.overall_sentiment` to drive the like/dislike toggle. In JavaScript, a missing key deserializes to `undefined` (not `null`), which silently breaks the `=== 'positive'` equality mapping. This plan guarantees the key is always present in both the OpenAI and fallback paths.

Output: Two new RED-first backend tests, then `overall_sentiment` wired into `_build_prompt` (schema line + rule) and `_fallback_interpretation` (default key + keyword matching).
</objective>
