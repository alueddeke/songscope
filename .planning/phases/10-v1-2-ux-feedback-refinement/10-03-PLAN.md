| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-03a | Tampering | overall_sentiment → aiSyncedFeedback | accept | Value is consumed only via strict equality (`=== 'positive'` / `=== 'negative'`); any other/malformed value is a no-op (toggle unchanged). | T-10-03b | Denial of Service | songscope:new-gem listener leak | mitigate | EVOLVE-02 uses `removeEventListener` in the useEffect cleanup (Task 3 acceptance gate asserts its presence) — prevents listener accumulation / duplicate-fetch amplification on re-mount. CustomEvent carries no payload, so no injection surface. |
| T-10-03c | Spoofing | global CustomEvent name | accept | Any script on the page could dispatch `songscope:new-gem`; the only effect is a harmless idempotent GET refetch of the user's own metrics. No state mutation, no privileged action. Acceptable for a single-user portfolio app. |
| T-10-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this plan (RESEARCH.md "no new dependencies"). No install task exists; no checkpoint required. |
