Sources with few trials have wider distributions, more exploration. Formula: `theta_i ~ Beta(s_i + 1, f_i + 1)`, then `source = argmax_i theta_i`. Code: paste the relevant block from `get_recommendation_weights` (lines 89-142). Talking point: "Thompson Sampling over epsilon-greedy because it is parameter-free and naturally handles the cold-start (every source starts at Beta(1,1) = uniform)."

    4. `## Online Learning (SGD on Taste Vector)` — Intuition: each like nudges the taste vector toward the liked track's genres; each dislike nudges away. Mirrors stochastic gradient descent on a preference function. Formula: `v[g] := v[g] + lr * signal` for each genre `g` in the track, where `signal = +1` for like, `-1` for dislike, `lr` is the learning rate from personalization_engine. Code: actual `apply_feedback_learning` snippet — verify the learning rate constant from the codebase, not a guess. Talking point: "Online learning means the model never needs a batch retrain — every feedback updates the same dict immediately. Aligns with the single-user constraint."

    5. `## Jaccard Diversity` — Intuition: how different is the set of genres I have recommended? Pairwise overlap between any two recommendations, averaged. Formula: `J(A,B) = 1 - |A ∩ B| / |A ∪ B|`. Code: paste `_jaccard_distance` from `backend/apps/core/views.py` (the function added in plan 04-02).
